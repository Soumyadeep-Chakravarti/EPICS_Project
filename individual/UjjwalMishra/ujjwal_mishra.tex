    \textbf{\Large Abstract}

\noindent Breast cancer continues to be a major health concern worldwide, and catching it early can save lives. Our team built a machine learning system that not only detects cancer accurately but also explains its reasoning -- important for any medical application where doctors need to trust and understand the output.

\vspace{0.5cm}
\noindent We tested out four different classifiers -- Logistic Regression, Decision Trees, SVM, and Random Forest -- and combined them with soft voting. My job was specifically implementing these individual algorithms and figuring out which configurations worked best.

\vspace{0.5cm}
\noindent We focused heavily on making the system interpretable, with confidence scores and feature importance analysis. In the end, Logistic Regression and SVM both hit 98.25\% accuracy with ROC-AUC scores above 0.99 -- pretty impressive results for classical ML methods.

\vspace{0.5cm}

\subsection{Overview}
My main job on this project was \textbf{implementing the machine learning algorithms} we used for classification. I got to work hands-on with four different classifiers, tuning them and figuring out what worked best for our breast cancer detection problem.

\subsection{Algorithm Implementation}

I implemented four classification algorithms. Here's how each one went:

\subsubsection{Logistic Regression}
This was our baseline -- simple but effective. It models the probability of malignancy using a sigmoid function:

\begin{equation}
P(y=1|x) = \sigma(w^T x + b) = \frac{1}{1 + e^{-(w^T x + b)}}
\end{equation}

What I actually set up:
\begin{itemize}
    \item Added L2 regularization (C=1.0) to prevent overfitting
    \item Bumped max\_iter to 5000 because it wasn't converging with the default
    \item Used LBFGS optimizer (works well for smaller datasets)
    \item Hit 98.25\% accuracy -- honestly surprised how well this simple model did
\end{itemize}

\begin{lstlisting}[style=pythonstyle, caption={Logistic Regression Implementation}]
from sklearn.linear_model import LogisticRegression

lr_model = LogisticRegression(
    max_iter=5000,
    C=1.0,
    random_state=42
)
lr_model.fit(X_train, y_train)
\end{lstlisting}

\subsubsection{Decision Tree Classifier}
I included Decision Tree mostly for interpretability -- you can actually trace through its decisions:

\begin{itemize}
    \item Used Gini impurity for splits (standard choice)
    \item Let it grow to full depth to see what it could do
    \item Only got 91.23\% accuracy -- definitely the weakest performer
\end{itemize}

The Decision Tree's rules are easy to understand, but it struggled compared to other methods. Probably overfitting on training data.

\subsubsection{Support Vector Machine (SVM)}
SVM with RBF kernel was interesting to implement. The kernel maps data to higher dimensions:

\begin{equation}
K_{RBF}(x_i, x_j) = \exp\left(-\gamma \|x_i - x_j\|^2\right)
\end{equation}

My configuration:
\begin{itemize}
    \item RBF kernel (handles non-linear boundaries if they exist)
    \item C=1.0 for regularization
    \item Had to enable probability=True for the ensemble to work
    \item gamma='scale' lets sklearn pick a reasonable value automatically
    \item Tied with Logistic Regression at 98.25\%
\end{itemize}

\begin{lstlisting}[style=pythonstyle, caption={SVM Implementation}]
from sklearn.svm import SVC

svm_model = SVC(
    kernel="rbf",
    C=1.0,
    probability=True,
    random_state=42
)
svm_model.fit(X_train, y_train)
\end{lstlisting}

\subsubsection{Random Forest Classifier}
Random Forest is basically a bunch of decision trees voting together:

\begin{equation}
\hat{y} = \text{mode}\{h_1(x), h_2(x), ..., h_B(x)\}
\end{equation}

where each $h_b(x)$ is one tree's prediction.

What I configured:
\begin{itemize}
    \item 100 trees (n\_estimators=100) -- standard starting point
    \item Bootstrap sampling for diversity
    \item No depth limit -- let trees grow fully
    \item Got 95.61\% accuracy -- decent but not top tier
\end{itemize}

\begin{lstlisting}[style=pythonstyle, caption={Random Forest Implementation}]
from sklearn.ensemble import RandomForestClassifier

rf_model = RandomForestClassifier(
    n_estimators=100,
    random_state=42
)
rf_model.fit(X_train, y_train)
\end{lstlisting}

\subsection{Algorithm Analysis}

I spent some time thinking about why the results turned out this way:

\subsubsection{Why Logistic Regression Excelled}
\begin{itemize}
    \item Suryanshi's preprocessing (especially scaling) really helped -- linear models love normalized data
    \item Turns out the classes are pretty linearly separable after all
    \item The regularization prevented overfitting on our 455 training samples
\end{itemize}

\subsubsection{Why SVM Performed Well}
\begin{itemize}
    \item RBF kernel can find non-linear boundaries, but honestly didn't need to here
    \item 30 features is a nice sweet spot for SVM -- not too few, not too many
    \item Margin maximization gives robust classification
\end{itemize}

\subsubsection{Why Decision Tree Underperformed}
\begin{itemize}
    \item Without pruning, it memorized the training data (overfitting)
    \item Small changes in data lead to completely different trees
    \item Can't capture smooth boundaries -- everything is axis-aligned splits
\end{itemize}

\section{PROJECT OVERVIEW}

\subsection{Problem Statement}
Our goal was to build a system that distinguishes benign from malignant tumors, provides confidence scores, and explains its decisions. In healthcare, you can't just have a black box -- doctors need to understand and trust the reasoning.

\subsection{System Architecture}
We organized the project into layers:
\begin{enumerate}
    \item \textbf{Data Layer}: Loading and preprocessing
    \item \textbf{Model Layer}: The classification algorithms (my focus area)
    \item \textbf{Ensemble Layer}: Soft voting combination
    \item \textbf{Output Layer}: Metrics and visualizations
\end{enumerate}

\subsection{Results Achieved}
Final results were solid:
\begin{itemize}
    \item LR and SVM both hit 98.25\% accuracy
    \item ROC-AUC above 0.99 for the best models
    \item 98.61\% recall -- catching almost all actual cancer cases
    \item Cross-validation confirmed these weren't fluke numbers
\end{itemize}

\section{CONCLUSION}

My job on this project was implementing the core ML algorithms -- Logistic Regression, Decision Tree, SVM, and Random Forest. I got to experiment with different configurations and figure out what worked best for our cancer detection problem.

The results were honestly a bit surprising. I expected the more complex methods (SVM, Random Forest) to dominate, but simple Logistic Regression tied for the best performance. It taught me that fancy algorithms aren't always better -- sometimes clean data and a simple model is all you need.

Each algorithm has its place though. Logistic Regression and SVM gave us the best accuracy, but Decision Tree's rules are easy to explain to non-technical people. Random Forest gave decent performance with built-in feature importance. They all contributed something to the final ensemble.

This EPICS project gave me real hands-on experience with ML implementation. It's one thing to learn about algorithms in class -- it's another to actually build them, tune them, and see why they behave the way they do. That practical experience is going to be valuable going forward.
